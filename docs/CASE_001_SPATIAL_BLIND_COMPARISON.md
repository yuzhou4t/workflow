# 空间面板盲测对比报告：HypoWeaver-Qwen、Agent Laboratory 与原论文

## 1. 结论先行

本次测试证明了两件不同的事：

1. **方法选择已经能够自然对齐。** 两套系统在没有看到原论文方法名称、模型方程和结果的情况下，都根据“本地直接关联、跨地区间接关联、总关联”三个估计目标，以及空间面板和权重资产，自主选择了双向固定效应空间杜宾模型（SDM）。这说明 `DesignEnvelope + 多候选设计 + Probe + Reviewer` 能把方法引向原文研究范式，而不必泄露“请使用 SDM”。
2. **数值尚不能称为复现。** 原论文使用 31 省和作者的试点区距离矩阵；本次安全案例只有 30 省，并使用公开省会坐标按披露公式独立重建的全省距离矩阵，部分控制变量也只是近似映射。因此 HypoWeaver 与原文结果方向和显著性不同，主要反映输入资产不同，不能据此判断哪个结论更正确。

在相同安全输入下，HypoWeaver 完成了真实空间估计、独立复算、科学审计、H3 结论降级、8 章写作和 H4 封存；Agent Laboratory 的研究计划同样选择了 SDM，但运行环境缺少 `spreg`，生成代码自动退化为把内生空间滞后因变量当作外生变量的 OLS。Agent Laboratory 随后的解释节点正确判定该结果无效并拒绝形成实证结论。

因此，本案例当前适合评价**方法识别、失败处理、审计与写作约束能力**，暂不适合评价精确系数复现能力。

## 2. 测试边界与盲测隔离

两套系统共享同一个安全可见案例：

- 2014—2023 年、30 个省份、300 个省份—年份观测；
- `lnSD`、`zGF`、控制变量及中立变量字典；
- 独立重建并冻结哈希的空间权重矩阵；
- 估计目标：区分直接、间接和总空间关联；
- 结论边界：只允许关联性解释。

两套系统运行前均不可见：

- 原论文 PDF；
- 原论文使用的方法名称与方程；
- 原论文表格、系数、显著性和结论；
- 作者代码与隐藏参考摘要。

只有 HypoWeaver Run 和 Agent Laboratory Run 均结束后，才打开隐藏参考进行本报告中的对照。隐藏参考没有回写任何 Run，也没有触发重新选模型或重跑到显著。

## 3. 三方方法对比

| 维度 | HypoWeaver-Qwen | Agent Laboratory | 原论文 |
|---|---|---|---|
| 方法家族 | 双向固定效应 SDM | 计划为双向固定效应 SDM | 空间—时间双向固定效应 SDM |
| 方法如何产生 | 3 个候选设计，经 Probe、4 类 Reviewer 和 H2 人工选择 | 单条 LLM 计划 | 作者研究设计 |
| 样本 | 30 省，2014—2023，N=300 | 同一安全案例，N=300 | 31 省，2014—2023，N=310 |
| 结果变量 | `lnSD` | `lnSD` | `lnSD` |
| 核心解释变量 | `zGF` | `zGF` | `zGF` |
| 控制变量 | `zFDI, EPD, UR, US, ER` | 同左 | `zINoD, EPD, UR, US, ERI` |
| 权重矩阵 | 公开坐标重建的全省 Haversine 距离矩阵 | 同左 | 作者的试点区距离矩阵 |
| 因变量空间滞后 | 真正作为内生空间项估计 | 生成代码因缺库而错误地当作外生项 | 正式 SDM 估计 |
| 固定效应 | 省份 + 年份 | 生成代码做双向去均值 | 省份 + 年份 |
| 推断 | 条件协方差与 rho 曲率的近似 Delta；非聚类稳健 | OLS 标准误；系统自行判定无效 | 31 省聚类稳健标准误 |
| 效应分解 | 直接、间接、总效应 | 因估计无效而未形成有效分解 | 直接、间接、总效应 |
| 模型选择诊断 | 已规划但未执行 | 已规划但未执行 | LR/Wald 等检验支持 SDM |
| 独立复算 | 有，`1e-8` 数值容差内匹配 | 无 | 隐藏材料未提供可直接复算的完整空间资产 |

方法家族层面的对齐是成功的；估计资产和推断实现层面的对齐仍不充分。

## 4. 三方结果对比

### 4.1 核心空间结果

| 指标 | HypoWeaver-Qwen | Agent Laboratory | 原论文 |
|---|---:|---:|---:|
| 直接效应 | -0.000713，p=0.850 | 未得到有效直接效应 | 0.101，SE=0.045，5% 显著 |
| 间接效应 | -0.023136，p=0.574 | 未得到有效间接效应 | 0.522，SE=0.265，5% 显著 |
| 总效应 | -0.023849，p=0.576 | 未得到有效总效应 | 0.623，SE=0.268，5% 显著 |
| 空间自回归参数 rho | 约 -0.99，边界警告 | 未一致估计 | 0.462，SE=0.104，1% 显著 |
| 科学判断 | `limited`，证据不充分 | 无结论，估计无效 | 正向本地效应与正向空间溢出 |

Agent Laboratory 的 OLS 输出中 `zGF=-0.000866 (p=0.849)`、`W_zGF=-0.047687 (p=0.269)` 只是有偏代码的中间产物，不是 SDM 的有效直接和间接效应，因此没有把这些数值放入可比效应栏。

### 4.2 为什么不能把数值差异直接归因于系统能力

本次安全案例与原论文至少存在四项实质差异：

1. 原论文为 31 省、310 个观测，本次为 30 省、300 个观测；
2. 原论文使用试点区距离矩阵，本次为按公开坐标和披露距离公式重建的全省矩阵；
3. 原论文控制变量为 `zINoD` 和 `ERI`，本次只能使用近似映射的 `zFDI` 和 `ER`；
4. 原论文使用省级聚类稳健推断，本次 HypoWeaver 仍采用近似 Delta 推断。

尤其是空间矩阵改变会同时改变 `rho`、结构系数和效应分解。当前 `rho≈-0.99` 本身已经提示重建矩阵与数据组合可能处在不稳定边界。因此，本次与原文的方向差异应登记为“**不可归因的复现差异**”，不能包装成对原论文的反驳，也不能通过继续换模型追求原文显著性。

## 5. 流程与写作能力对比

| 能力 | HypoWeaver-Qwen | Agent Laboratory |
|---|---|---|
| 候选方案 | 3 个独立目标候选 | 1 个计划 |
| 前置可行性检查 | 每候选无结果 Probe | 数据画像与计划自检 |
| Reviewer | Measurement、Causal、Statistical、Reproducibility 隔离审查 | 无独立 Reviewer Arena |
| 人工冻结 | H2 选择候选并冻结合同与资产哈希 | 无同等级冻结闸门 |
| 执行与科学状态 | 分开记录 | 顶层仍为 `execution_status=success, scientific_status=not_assessed`，报告内部再判无效 |
| 复现 | 第二次独立执行并逐字段比较 | 无 |
| 结论约束 | H3 逐条授权和降级 | 解释节点整体给出“无结论” |
| 写作 | 8 章、约 10,247 字、v7、一致性审计通过 | 7 部分短报告，因估计无效而拒绝实证结论 |
| 最终人工审稿 | H4 共 5 次定点退回后批准 | 无 |
| 封存 | 合同、运行、Claim、稿件哈希和 HMAC 封存 | 产物文件保留，无同等级 Claim/稿件封存 |

HypoWeaver 的优势不在于“更容易得到显著结果”，而在于能够把失败、受限证据和人工修改完整地记录下来，并阻止未经授权的控制变量系数、因果措辞和未执行检验进入最终稿。Agent Laboratory 的优点是链路短、成本低、失败解释清楚；其主要短板是执行代码可以在缺依赖时自行退化，而顶层状态仍把“代码返回 0”记为执行成功。

## 6. 对当前 Group 2 工作流的判断

### 已经符合《框架设计》的部分

- 通过多个候选扩大研究设计空间，而不是由单一 Agent 一次定稿；
- Probe 在查看结果前检查数据、测量、识别和执行条件；
- Reviewer 以不同职责独立攻击候选，不用平均分或多数投票决定科学真值；
- H2 冻结正式研究合同，执行后不得按显著性静默换模型；
- 把代码执行成功、科学有效性和假设是否得到支持分成三个判断；
- 主运行后独立复算，并用 ClaimLedger 与 H3 控制结论强度；
- Writer 之后设置 H4，人类可按章节定点退回。

### 尚未达到完整目标态的部分

- 当前 Probe 是结构化可行性检查，还不是多分支、可执行、带预算的 Probe Tree；
- 4 个 Reviewer 虽然上下文隔离，但仍属于同一 Qwen 模型家族，存在相关偏差；
- G2 候选选择和 G3 Probe 后冻结在第一版中合并进 H2；
- 尚未接入 Group 1 的 EvidenceBundle、知识图谱和 Citation Auditor；
- 执行器仍只覆盖少数方法，不能把计划中的诊断、稳健性和证伪检验全部执行；
- H4 本次需要 5 轮人工定点退回，说明自动写作审计仍有误报和漏报成本。

当前做法符合 Group 2 Task 3 的第一版边界。此时扩成 19 个动态角色并不会自动提高研究质量；应先用多个案例证明 Probe、Reviewer、冻结合同和复现各自带来的增量。

## 7. 下一步计划：既自然贴近原文，又不泄露答案

### P0：把案例分成三种 Benchmark 轨道

1. **发现盲测 `discovery_blind`**
   只给研究问题、假设、完整中立变量字典和数据，让系统自主选变量、方法和检验。评价“研究自主性”。
2. **方法对齐盲测 `reproduction_aligned`**
   额外给中性估计目标、必须区分的效应、识别约束和合法数据资产，但不提供方法名称、方程、系数方向或显著性。评价“能否自然走到与原文可比的方法”。本次空间案例属于这一轨道。
3. **精确复现 `exact_replication`**
   提供作者公开的确切分析样本、变量构造规则、空间权重或政策处理资产、软件版本和随机种子；只隐藏结果表和结论。评价数值复现能力。

三种轨道必须分开报告，不能用一个总分混淆“自主发现”和“忠实复现”。

### P0：建立案例可复现等级

- **A 级**：原始分析样本、变量定义、处理/权重资产、代码或完整规格都可得，可做数值误差评分；
- **B 级**：方法目标和大部分数据可得，但关键权重、样本或变量存在缺口，只做方法结构评分；
- **C 级**：只有论文和不完整数据，只能测试研究设计与缺口识别。

本次空间案例应标为 B 级。在取得作者实际权重矩阵、固定 31 省顺序和精确控制变量前，不进入数值复现排行榜。

### P0：补齐空间执行器的科学闭环

1. 实现或接入可验证的固定效应 SDM 一致估计器；
2. 增加省级聚类稳健或经过验证的空间推断；
3. 执行 Moran、LM、LR/Wald 和固定效应规格检验；
4. 执行冻结的替代权重、时间趋势和证伪检验；
5. 当 `rho` 到达稳定区间边界时，自动判定为需要修订或停止，而非继续形成普通结论；
6. 禁止缺依赖时把内生空间项退化为 OLS 后仍标记“执行成功”。

### P1：增强 Reviewer 与智能体博弈，但保持边界

1. Reviewer 输出统一为“问题—证据—严重度—必须修改内容”，禁止给候选打主观总分；
2. 对高风险案例抽取一名不同模型或确定性统计审计器做交叉审查，测量同模型相关偏差；
3. 把 Probe 升级为最小可执行 Probe：只验证字段、矩阵、收敛和估计器能力，不读取或比较主结果显著性；
4. 保存 Reviewer 分歧矩阵，交给 H2 人类选择，不让 Arena 自动决定科学真值；
5. 用消融实验分别移除 Probe、Reviewer、H2 冻结、独立复现和 H3 白名单，验证每一层是否真的提升质量。

### P1：三案例校准顺序

1. **企业面板固定效应案例**：优先使用数据、变量、作者代码和参考表完整的案例，先把精确数值复现链路校准稳定；
2. **政策评估 DID 案例**：在 DID 执行器具备平行趋势、动态效应和处理定义校验后再运行；
3. **空间面板案例**：先作为方法对齐压力测试，取得精确权重与 31 省样本后再升级为数值复现案例。

## 8. 下一轮统一验收指标

每个案例至少报告：

- 估计目标覆盖率；
- 变量角色、样本规则、处理定义或权重资产一致率；
- 方法结构与原文的对齐程度；
- 主运行与独立复算是否一致；
- 系数、标准误、样本量和显著性与原文的差异；
- 诊断、稳健性、证伪和识别步骤完成率；
- Reviewer 对专家问题的召回率与误报率；
- 未经 H3 授权进入稿件的统计量数量；
- 人工闸门次数、模型调用量和总耗时；
- 所有差异是否能归因到数据、规格、执行器或随机性。

只有输入资产达到 A 级且上述差异可以逐项归因，才使用“复现成功/失败”；否则使用“方法对齐”“部分复现”或“不可判定”。
