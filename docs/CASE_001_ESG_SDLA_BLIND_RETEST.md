# 企业面板盲测复测报告：HypoWeaver-Qwen、Agent Laboratory 与原论文

> 测试日期：2026-07-15
> 案例：`case_001_企业ESG表现与短债长用`
> 盲测轨道：`reproduction_aligned`（方法对齐盲测，不是完全自由发现盲测）

> 版本说明：本文保留的是执行器扩展前的历史复测，用于记录两套系统当时的差异。当前企业面板执行器已经补齐单例处理、推断口径及冻结的诊断/稳健性/证伪/交互步骤；最新终态见 [`cases/case_001_esg_sdla/FINAL_EXECUTION_REPORT.md`](cases/case_001_esg_sdla/FINAL_EXECUTION_REPORT.md)。

## 1. 结论先行

本轮选择该案例，是因为它同时具备完整企业面板数据、中立案例说明、变量字典、原论文、作者 Stata 代码、原始结果表和附录，且基准模型落在当前 Python 执行器已经真实支持的范围内。相比缺少精确权重矩阵的空间案例，它更适合校准“从假设到建模、执行、审计和论文写作”的完整链路。

本次复测得到四个主要结论：

1. **两套系统的基准结果完全一致，并与原文主系数高度一致。** HypoWeaver-Qwen 和 Agent Laboratory 均得到 `β(ESG_w)=-0.151558`；原论文报告 `-0.152`。方向、数量级和四舍五入后的数值一致。
2. **这仍是“部分复现”，不是完整复现。** 两套系统使用 30,311 条观测，原文表 3 使用 29,919 条。数据中恰有 392 家只出现一次的企业，删除这 392 条单例观测后正好得到 29,919；这与作者使用的 `reghdfe` 默认删除单例组完全吻合。标准误和 R² 的口径也尚未与 Stata 完全对齐。
3. **HypoWeaver 的证据边界更可靠，但执行广度不足。** 它只把真实执行的基准模型写成结论，H2—H4 机制检验因未执行而保持 `hold/prohibited`；最终生成 8 节、约 9,330 字符的论文初稿并通过一致性审计。缺点是冻结计划中的稳健性、证伪和机制步骤尚未由执行器真正运行。
4. **Agent Laboratory 的执行广度更大，但科学审计不足。** 它自动生成并运行了基准、替代变量、动态和机制代码，部分稳健性数值也与原文一致；但机制估计形式与原文不一致，报告中仍出现固定效应系数的企业间解释、因果式单位表述和 R² 误读，顶层科学状态仍为 `not_assessed`。

另外，先前失败**不是 API 额度耗尽**：千问最小真实调用返回 HTTP 200，失败日志为长请求连接被关闭或生成代码运行错误；没有出现 `429`、`insufficient_quota` 或余额不足响应。

### 3.1 样本量差异已经定位

当前 CSV 包含 30,311 条观测、3,764 家企业，`S + YEAR` 无重复。其中：

```text
仅出现 1 次的企业：392 家
对应单例观测：392 条
30,311 - 392 = 29,919
```

作者代码使用：

```stata
reghdfe SDLA_w ESG_w SIZE_w LEV_w ROA_w GROWTH_w CF_w FA_w BOARD_w TOP1_w SOE_w MH_w, \
    absorb(S YEAR INDUSTRY) vce(cluster S)
```

`reghdfe` 默认删除无法贡献组内识别的单例组。精确相等的样本数说明，当前 Python 执行器保留单例企业，是与原文样本口径不同的首要原因。单例并不改变组内系数的有效识别信息，因此两套系统仍能得到与原文相同的主系数；但它会影响报告的 N、自由度修正和聚类标准误。

### 3.2 R² 不能直接横向比较

系统当前输出的是 PanelOLS 的 **Within R²**；原文表格中的 `0.629` 来自 Stata 高维固定效应回归的表格口径。二者不是同一统计量。Agent Laboratory 把 `Within R²=0.0603` 解释为“大部分变异由个体固定效应解释”也不严谨：Within R² 只描述去除个体效应后的组内拟合，不能据此分解固定效应解释了多少总变异。

下一版执行器必须同时标记 `within / between / overall / adjusted` 等口径，并禁止跨口径评分。

最终成功 Run 共 6 次 LLM 调用，输入 91,777 tokens、输出 10,058 tokens，工作流耗时 195.98 秒；生成报告约 4,584 字符。前两次失败说明基线仍受模型生成代码随机性影响，第三次成功不能掩盖其稳定性问题。

### 5.2 执行广度

Agent Laboratory 除基准模型外还执行了：

- 替换 ESG 评级：`-0.006971，p<0.001`；原文为 `-0.007***`；
- 替换结果变量 SLEV：`-0.032733，p=0.0262`；原文为 `-0.033**`；
- ESG 领先/滞后动态检验；领先项显著，系统据此警告反向因果或前定趋势；
- 信息透明度、长期机构投资者和过度投资的顺序回归。

它在替代测量层面表现很好：两个稳健性系数均与原文的四舍五入结果一致。这证明数据与基准代码生成能力具备较好的数值基础。

### 5.3 机制结果不能直接与原文比较

Agent Laboratory 把 H2—H4 理解为中介路径，采用“ESG→机制变量，再把机制变量加入 SDLA 回归”的顺序回归；原论文实际检验的是机制变量与 ESG 的**交互项**：

- `DA1 × ESG`、`DA2 × ESG`、`Restate × ESG`；
- `LTINS × ESG`、`STINS × ESG`；
- `OVERINV × ESG`。

因此，Agent Laboratory 的“长期机构投资者部分中介”“H3 得到支持”等判断与原文不是同一估计量，不能计为机制复现。过度投资检验还损失 66.59% 样本，系统将其判为无效，这一失败处理是诚实的。

### 5.4 报告质量问题

Agent Laboratory 报告具有表格和更多实证内容，但存在三类明显风险：

1. 在企业固定效应模型下写成“ESG 表现较高的企业短债长用更低”，混入企业间比较；
2. 写成“ESG 每提高一个单位，SDLA 平均降低”，在只允许关联解释时语气过强；
3. 把 Within R² 低解释为“大部分变异由个体固定效应解释”，超出了该统计量本身的信息。

报告虽主动披露“无 ClaimLedger、无独立科学审计”，但顶层仍为 `execution_status=success, scientific_status=not_assessed`。代码跑通与科研结论有效之间尚缺确定性闸门。

Agent Laboratory 最终报告保存在本机被 Git 忽略的 Benchmark 运行目录中，对应 Run 为 `baseline-9b97db3e-7d1a-4464-b8c1-d73e88d1937f`；公开仓库不包含该运行产物。

## 8. 如何让方法与原文更自然对齐，同时不泄露答案

### P0：先对齐执行器的计量口径

1. 对固定效应复现轨道默认检测并删除单例企业，同时把删除数量写入 deviation log；
2. 对齐 Stata/reghdfe 的聚类自由度与小样本修正；
3. 同时输出并明确命名 Within、Between、Overall 与调整后 R²；
4. 对企业、年份、行业固定效应逐项记录“实际吸收、因共线性被吸收、或未执行”；
5. 为相同冻结合同增加 Python 与 Stata/R 的交叉执行器测试。

### P0：让执行器真正遍历冻结计划

当前 HypoWeaver 的计划已经包含诊断、替代结果和机制边界，但执行器只运行 `baseline_models`。下一步应按冻结顺序执行：

```text
sample_rules
→ variable_construction
→ baseline_models
→ diagnostics
→ robustness_tests
→ falsification_tests
→ mechanism_tests
→ heterogeneity_tests
```

未实现的方法必须明确返回 `unsupported`，不能用基准模型代替，也不能由 Writer 把“计划”写成“结果”。

### P0：补齐 132 个字段的中立元数据

当前安全字典只解释了约 23 个核心字段。系统看得到其余列名，却不知道它们的构造、时间顺序、角色和适用边界，因此很难自然发现原文的工具变量、异质性分组和经济后果分析。

应为所有可见候选字段补充：

- 中立语义和计算公式；
- 数据来源与时间可用性；
- `outcome / exposure / control / moderator / candidate_instrument / grouping / future_outcome` 等角色候选；
- 缺失率、支持样本和潜在泄露风险；
- 候选工具变量只提供理论相关性与排除限制待审查状态，不告诉系统原文估计结果。

元数据是“系统理解数据所必需的事实”，不是原文答案。它比提示词直接写“请做 IV/Heckman”更自然，也更适合跨案例复用。

### P1：把机制假设写成可辨识的估计目标

本轮“可能构成传导渠道”的措辞过于模糊，Agent Laboratory 自然地选择了顺序中介回归，而原文检验的是调节/交互效应。方法对齐轨道可改成：

> 检验 ESG 与 SDLA 的条件关联是否随信息透明度、机构投资期限结构或投资决策质量而变化；由系统选择满足数据与识别条件的交互、分组或其他条件效应估计方式。

这只明确**估计量类别**，不提供公式、系数或显著性，既能保留盲测，又能避免把“机制”误解为完全不同的统计任务。

### P1：将 Benchmark 分成三条轨道

1. `discovery_blind`：只给研究问题、完整中立字典和数据，评价自主研究能力；
2. `reproduction_aligned`：再给估计目标、识别边界和研究范式约束，评价能否自然走到可比方法；
3. `exact_replication`：提供作者精确样本规则、软件版本和完整规格，只隐藏结果，评价数值复现。

三条轨道应分别计分。若把方法对齐信息放进发现盲测，会高估自主发现能力；若在精确复现中不提供单例规则和软件口径，又会把环境差异错算成模型失败。

### P1：保持 Agent Laboratory 基线独立

不应把 HypoWeaver 的 Critic、冻结合同或 ClaimLedger 塞回 Agent Laboratory 内部。正确做法是：

- Agent Laboratory 保持原始 `plan → execute → interpret → write` 基线；
- 两套系统均终态后，由独立 Benchmark Evaluator 对输入隔离、方法对齐、数值、科学有效性和写作越界统一评分；
- 评测结果不得反馈给本次 Run 重新生成到“更像原文”为止。

这样既保留公平基线，也能看清 Reviewer Arena、H2 冻结、复算、H3/H4 究竟带来了多少增量。

## 9. 下一轮验收标准

下一轮企业面板复测至少应达到：

- 删除单例后主回归 N 精确等于 29,919；
- 主系数、聚类标准误和显著性与原文在预设容差内一致；
- R² 使用同口径比较；
- 冻结的替代 ESG、替代 SLEV 至少真实执行；
- 机制假设按条件效应/交互估计量执行，不再用顺序中介冒充；
- 未执行的 IV、Heckman、异质性和经济后果保持未执行，不进入结论；
- HypoWeaver 记录模型调用量和耗时，能够与 Agent Laboratory 比较成本；
- 两套系统终态后再打开隐藏参考，并生成机器可读的逐项差异表。
